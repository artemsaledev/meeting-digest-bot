# Пользовательская справка по Telegram-командам

`MeetingDigestBot` работает в Telegram-группе с постами Loom-дайджестов от `AIcallorder`.
Основной сценарий: ответить на нужный пост, упомянуть бота и написать команду.

```text
@LLMeets_bot preview
```

Если команда отправляется не reply-сообщением, можно вставить ссылку на пост прямо в текст команды.

```text
@LLMeets_bot https://t.me/c/<chat>/<message> preview
```

## Команды по встречам

| Команда | Когда использовать | Что делает |
|---|---|---|
| `@LLMeets_bot помощь` | Нужна краткая справка в Telegram | Показывает примеры команд. |
| `@LLMeets_bot preview` | Перед записью в CRM | Показывает, что бот понял из поста, какую задачу нашел и что сделал бы в auto-режиме. В CRM ничего не пишет. |
| `@LLMeets_bot создать` | Нужно создать новую CRM-задачу по встрече | Создает задачу в проекте `Bitrix Develop Team`. Добавляет описание, ссылки на Loom/Google Docs и чек-листы QA/PM. |
| `@LLMeets_bot коммент 166229` | Нужно добавить итоги встречи в существующую задачу | Добавляет комментарий с результатами встречи: резюме, решения, обязательства, блокеры, техдолг, запросы на оценку и ссылки. |
| `@LLMeets_bot чеклист 166229` | Нужно добавить только чек-листы | Добавляет QA и PM чек-листы в существующую задачу. Повторяющиеся пункты дедуплицируются по тексту. |
| `@LLMeets_bot обновить 166229` | Нужно развивать одну постановку несколькими встречами | Не перетирает описание. Добавляет новую встречу как следующий `Point N`, учитывает старое описание как контекст, добавляет отдельный чек-лист и комментарий. |
| `@LLMeets_bot зарегистрировать` | Старый пост не найден ботом | Регистрирует старый digest-пост по reply. После этого можно снова выполнить `preview`, `создать`, `коммент`, `чеклист` или `обновить`. |

## Как работать со старым постом

Если бот отвечает, что публикация не зарегистрирована:

1. Ответьте командой `@LLMeets_bot зарегистрировать` именно на старый пост с Loom-дайджестом.
2. Дождитесь ответа `Старый пост зарегистрирован`.
3. Ответьте на тот же пост нужной командой, например `@LLMeets_bot обновить 168334`.

Бот извлекает из старого поста Loom URL / video ID, заголовок встречи, Google Doc, Transcript Doc и названия секций документа.

## Команды для плана дня

Daily-план собирается только из встреч Loom, в названии или digest-артефактах которых есть `#daily`.

| Команда | Что делает |
|---|---|
| `@LLMeets_bot план 2026-05-04 preview` | Показывает, какая задача плана дня будет создана и какие чек-листы появятся. |
| `@LLMeets_bot план 2026-05-04 создать` | Создает задачу `План дня DD.MM.YYYY / Bitrix Develop Team`. |
| `@LLMeets_bot план 2026-05-04 создать команда Bitrix Develop Team` | То же самое, но с явным названием команды. |

В задачу плана дня добавляются участники команды, а пункты чек-листа получают `MEMBERS` по распознанным ответственным.

## Команды проверки выполнения daily

Эти команды не переносят пункты на следующий день. Они только проверяют чек-листы задачи дня, собирают незакрытые пункты и добавляют отчет комментарием в задачу дня.

| Команда | Что делает |
|---|---|
| `@LLMeets_bot итоги вчера` | Проверяет задачу плана за вчера по Киеву. |
| `@LLMeets_bot итоги 2026-05-04` | Проверяет задачу плана за указанную дату. |
| `@LLMeets_bot итоги недели 2026-05-04 2026-05-08` | Собирает недельный отчет по daily-задачам в указанном диапазоне. |

В Telegram-ответе бот тегает ответственных за незакрытые пункты. В недельном отчете все ответственные с незакрытыми пунктами упоминаются одной строкой.

## Автоматические отчеты

На VPS настроен cron:

| Время | Действие |
|---|---|
| Каждый день `09:00 Europe/Kyiv` | Проверка daily-плана за вчера. |
| Каждую пятницу `16:00 Europe/Kyiv` | Недельный отчет по текущей рабочей неделе. |

Если дневная задача была удалена и не создана заново через бота, недельный отчет ее не найдет. Если задача была пересоздана через бота, новая привязка заменит старую и отчет возьмет актуальную задачу.

## Как проводить daily, чтобы бот корректно собрал план

Рекомендуемый формат проговаривания:

```text
Daily 2026-05-04. Команда Bitrix Develop Team.

Иван Карповец.
План на сегодня:
1. Проверить смену способа оплаты.
2. Подготовить демо.
Блокеры: жду ответ Игоря.

Михаил Конев.
План на сегодня:
1. Протестировать Payments Pro.
Блокеры: нет.
```

Правила:

- Называть полное имя и фамилию перед блоком человека.
- Идти по людям последовательно, не смешивать планы нескольких людей в одном блоке.
- Использовать фразы `План на сегодня`, `Блокеры`, при необходимости `Сделано вчера`.
- Формулировать пункты как конкретные действия на день.
- Не включать в план обсуждение формата daily, общие разговоры и чужие предложения без явного ответственного.

## Короткое сообщение в группу перед daily

```text
Коллеги, daily записываем для автоматического плана дня в CRM.
Идем по очереди: называю полное имя и фамилию, затем человек коротко говорит:
1. план на сегодня;
2. блокеры, если есть;
3. что нужно от команды.
Просьба формулировать пункты как конкретные действия на день, без длинного обсуждения внутри своего блока.
```
